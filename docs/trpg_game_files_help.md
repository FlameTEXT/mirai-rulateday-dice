## 文件说明

***当系统载入后，载入的第一个事件节点名称必定为1，详细看 event(事件)***

该文件格式为json文件，模组文件需要放置在MCL程序目录下的 `data/indi.eiriksgata.rulateday-dice/trpg-game` 文件夹里面

第一层结构为：

```json
{
  "name": "",
  "describe": "",
  "introduction": "",
  "author": "",
  "version": "",
  "importData": {},
  "option": {},
  "event": {}
}
```

|参数名|类型|填写说明|说明|
|----|----|----|----|
|name|string|必填|模组名称|
|describe|string|必填|模组描述|
|introduction|string|必填|简介|
|author|string|必填|作者名称|
|version|string|必填|版本|
|importData|Object|必填|导入数据，目前仅用于存储角色数据|
|option|Object|必填|选项数据|
|event|Object|必填|事件数据|

Object类型的参数，将会在下面单独详细介绍

## importData

目前该集合内仅有一个role，因此是属于固定格式。

数据格式参考:

```json
{
  "importData": {
    "role": {
      "player": {
        "name": "",
        "describe": "player属于玩家属性，但是在这里填写的数据是不会有任何作用的，建议是通过游戏进程来获取东西。作者可以通过开始的时候送一些物品。",
        "attribute": "",
        "consumables": ""
      },
      "system": {
        "describe": "系统数据",
        "attribute": "动机概率50必然发生100"
      },
      "艾莉莎": {
        "describe": "同行调查员之一",
        "attribute": "力量50体质50体型65敏捷70外貌75智力55意志60教育55幸运75会计60侦查80聆听70",
        "consumables": "生命11魔法6理智60"
      },
      "莫比乌斯": {
        "describe": "同行调查员之一",
        "attribute": "力量50体质50体型65敏捷60外貌65智力80意志60教育55幸运75生命11魔法6理智60神秘学70社会学70",
        "consumables": "生命11魔法6理智60"
      }
    }
  }
}
```

1. player该项在这里仅用作占用块，实际数值要通过 `.trpg-role-set` 指令设置。
2. system 一般用于存储一些概率变量，用于选项中的检测使用。
3. attribute 为属性值，属性一般用于做检测 例如D100 < 属性值，这里属性包含技能,后面option中会详细介绍。
4. consumables 消耗品，在这里的数值，都可以自定义的进行检测 > 或 <  或 = 你给定的数值，后面option中会详细介绍。
5. 当你需要额外增加角色数据时，可以在role集合内 继续添加 数个Map 以达到添加数个角色。

|参数名|类型|填写说明|说明|
|----|----|----|----|
|describe|string|非必填|一般用于该角色的描述。目前不用于系统数据使用。|
|attribute|string|必填|角色的属性和技能项。用于下面option(选项)中的检定。|
|consumables|string|必填|消耗品、衍生属性的存放。用于下面option(选项)中的检定。|


## option(选项)

该内容为事件选项，每个事件下方才会携带选项。

```json
{
  "option": {
    "B1001": {
      "text": "随机动机",
      "count": 1,
      "detection": {
        "role": "system",
        "attribute": "动机概率",
        "showDice": true,
        "success": {
          "text": "来自银行、抵押公司等等的债务使你的财务状况陷入了危机。在绝望之中你选择向高利贷者求助。你必须在这周结束时拿到现金，不然高利贷者就要让你吃点苦头。你只需要1000美金就能脱离苦海……",
          "nextNode": "2",
          "update": {
            "role": "player",
            "attribute": "身负债务=100,信誉=10",
            "consumables": "美元=20"
          }
        },
        "fail": {
          "text": "你和绑架犯西德尼·哈里斯，还有他的党羽，很早就认识了。你们曾经一起长大，但不是朋友。哈里斯曾经是你人生中的灾星。一朝被欺负，十年难抬头……从那以后，你过得还不错，哈里斯则每况愈下。现在胜负已分，是你比他强了。这就是一个向他复仇的机会，以法律的名义……",
          "nextNode": "2",

          "nextNode": "3",

          }

          "nextNode": "3"
        }
      }
    }

```

从上方的例子中可以看出，option 是一个Map `B1001`作为Key 这个编码是作为选项ID，用于在事件下方作绑定，这样可以提高复用率。

### option map

|参数名|类型|填写说明|说明|
|----|----|----|----|
|text|string|必填|该参数为选项的显示文本。|
|count|int|必填|选项剩余的选择次数，只有当选项次数>0的时候，玩家才可以选择该项，否则，将不给予显示与选择。|
|detection|Object|必填|检测结构，在里面的数据是属于判断内容。因为是必填项，你觉得不需要检测，可以在system角色中增加一个必然通过100的属性|
|showDice|boolean|选填|如果该参数不存在，那么将默认不会显示骰子的数值，如果数值为true 那么将会返回骰子的结果。|

### detection
***如果角色没有检定的`属性`或者`消耗品`，都一律视为0***

检定分为两种
1. attribute ：属性检定，该检定在一定程度上等同于coc7的检定，即生成一个百面骰，然后判断其数值是否小于 你拥有的数值，如果是则通过，否则为失败。
2. consumables ：消耗品检定，该检定为判断消耗品数值是否满你给定的数值。满足则通过，同上。

|参数名|类型|填写说明|说明|
|----|----|----|----|
|role|string|必填|需要检测的角色名，例如你需要检测玩家的属性或者消耗品，那么则填写player，代表玩家角色。如果你是想检测NPC，那么你就填写对应的`importData-role-`的NPC名称|
|attribute|string|选填(与consumables必然要存在一个)|需要检测的属性项，目前仅允许填写一项，属性包括人物的基础属性例如：力量敏捷幸运等9项目，此外还包括技能全部，该项是仅作为trpg中的coc7的属性技能检定判断，但是没有大成功和大失败的说法，只有成功和失败。 填写内容例子：敏捷|
|consumables|string|选填(与consumables必然要存在一个)|需要检测的消耗品，其中填写例子：`绷带>0` 则判断绷带的数值是否>0，如果>0则检定通过。其中的符号包括三种：>、<、=  |
|success|Object|选填|检定通过执行的结构|
|fail|Object|选填|检定失败执行的结构|

***如果一个detection中存在 `attribute` （属性）和 `consumables`（消耗品）则需要两项都通过检测后才会 属于 success***

注意：每个detection中都会包含 success 与 fail 集合项 除非你检测百分之一百 成功 或者失败，否则都需要填写两个结构，从而避免出错。

此外每个detection中，必然包含一个 attribute 或者 consumables 否则将会出错。

### success 与 fail

success与 fail是一样的集合结构。每一个success|fail都拥有一下的参数结构。

|参数名|类型|填写说明|说明|
|----|----|----|----|
|text|string|必填|当检测成功或者失败时，将会返回该文本内容。| |nextNode|string|跳转到下一个节点的ID| |update|Object|更新属性技能或者消耗品的结构|
|nextNode|string|必填|当执行该结构后，跳转到的事件节点ID|
|update|object|选填|如果你需要对属性技能或者消耗品数值做出调整，那么则需要写该集合结构|


### update

***一个update结构中，必然包含role项***

|参数名|类型|填写说明|说明|
|----|----|----|----|
|role|string|必填|需要做出更改操作的角色名|
|attribute|string|选填|属性技能数值更改，例如:力量+5，则效果为力量属性在原来的基础上增加5。其中符号包括：+、-、=。分别是增加、减少、等于三项操作|
|consumables|string|选填|消耗品数值更改，例如：绷带=3，则效果为绷带消耗品数值设置为3个，其中符号包括：+、-、=。分别是增加、减少、等于三项操作|

## event(事件)

json例子如下：
```json
{
  "event": {
    "node": {
      "1": {
        "text": "选择调查员动机，虽然本模组没有根据不同的动机作不同的选项抉择（因为如果要那样做的话，无疑是等同于工作量翻了好几倍）。这个模组作为样例，不再考虑更多的情形。下面生成的动机，仅作为你个人代入角色的一个目的，使你更有沉浸感。各类事件也仅会根据这几种动机指定固定的选项。按照普遍的规则书来说，当玩家的行动符合自己的动机的时候，通常会给予一些奖励。",
        "option": [
          "B1001"
        ]
      },
      "2": {
        "text": "1925年6月20日周六，贝宁顿的街道上充斥着各种新闻和谣言，这些消息无一不是和昨晚发生在森林边缘的枪战有关。街上的人们或高声或细语地谈论着警车、汽笛、霰弹枪以及逃跑的绑架犯们的事情。他们说着在那场枪战之中，三名绑架犯被击毙，而两名警察则中弹了——有的人说他们只是受伤，而有的人则是认为他们已经死了。镇上各处都贴着海报，上面说治安官正在组织寻找卢卡斯·斯壮的失踪女儿以及对逃亡匪徒的搜捕。为了能寻找回卢卡斯那失踪的女儿，并且抓住那些潜逃的犯罪党羽。在镇子之中的每一名成员都被邀请去了警察局的会面。根据传言，卢卡斯先生声明不管是谁，只要能安全救回他的女儿，他愿意给予5000美金的奖励。\n 身为调查员的你正在警察局参加通报会，詹金斯警长正在说明此次事件的一些情况：西德尼·哈里斯及其党羽绑架了卢卡斯·斯壮的女儿。哈里斯要求10000美元的赎金来赎回，并且定在昨晚在城外交货，事情似乎进展得十分顺利，然而西德尼他们变得焦躁不安，并且突然变卦，发生了枪战。场面一片混乱，双方都发射了子弹。在射击了两名警察后，哈里斯及其一名同伙抢走赎金逃跑了，他们逃进了森林，推断他们应该把简藏在里面。在警长发话完毕后，卢卡斯·斯壮接着说道：[除了给5000美元作为奖励外，只要参加搜查队的，我还可以额外给每人每天25美元，希望大家能尽快找到我的女儿]。",
        "option": [
          "B1002",
          "B1003",
          "B1004"
        ]
      },
      "8": {
        "text": "本模组只是简单的展示了一下模组的编写例子。该模组不是一个完整的故事剧情。",
        "end": true
      }
    }
  }
}
```

***注意：每局游戏必然从事件节点1 开始，因此模组的编写者一定要出现节点1。***

|参数名|类型|填写说明|说明|
|----|----|----|----|
|node|Map<string,string>|必填|节点表|
|text|string|必填|进入事件时，必然发送的文本|
|option|array|选填|此处填写每个事件下 携带的option(选项)，可供玩家选择|
|end|boolean|选填|是否标记为结束事件，如果当end存在且为true时，那么在发送完text的内容后，则退出游戏，并且清空数据|
